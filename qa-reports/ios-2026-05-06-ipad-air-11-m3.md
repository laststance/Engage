## Issues

### ISSUE-001: Data Backup cannot create backups

| Field | Value |
|-------|-------|
| **Severity** | High |
| **Category** | Functional |
| **Screen** | Settings -> Data Backup |
| **State** | Default portrait |
| **Affected build** | 1.0.0 / local Release simulator build |

**What's wrong:** Tapping `Create Backup` shows `Backup Failed`. The error says `getInfoAsync` from `expo-file-system` is deprecated and must use the new `File`/`Directory` API or import legacy APIs from `expo-file-system/legacy`.

**Repro:**

1. Open Stats -> Settings -> Data Backup.
2. Tap `Create Backup`.
3. Observe the `Backup Failed` alert.

Evidence:

- `14-backup-settings.png`
- `15-backup-created.png`

Code locations:

- `src/services/backupService.ts:1`
- `src/services/backupService.ts:111`
- `src/services/backupService.ts:536`

Official doc checked:

- Expo FileSystem docs: https://docs.expo.dev/versions/v54.0.0/sdk/filesystem/

### ISSUE-002: Dark Mode is not visually honored

| Field | Value |
|-------|-------|
| **Severity** | Medium |
| **Category** | State |
| **Screen** | Data Backup sampled; likely broader |
